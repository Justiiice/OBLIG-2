Git

GitHub

GitHub er et nettsted hvor man enkelt kan samarbeide på et programeringsprosjekt samt kontrollere hvilke endringer som blir gjort.
Hovedelementene i GitHub er "repositories", "branches", "commits", og "Pull Requests" samt "Merging".
Man starter et Repository, legger inn basiskode, og om man vil endre eller legge til noe må man lage en branch som inneholder endringene og ofte gi denne branchen et navn.
Når du er sikker på at du vil trå til disse endringene så må man trykke commit. Endringene er enda ikke implementert i basiskoden enda!
Deretter trykker man på Pull Request for å foreslå disse endringene til koden. Nå kan du se mer spesifikt hvilke endringer du har gjort.
Etter du har lagd en Pull Request kan man Merge Request for å sammenslå denne branchen med basiskoden.
Dette gjør det veldig enkelt å se tilbake på hvilke endringer som blir gjort når, hvor og av hvem. I et samarbeidsprosjekt er det veldig vanskelig å holde styr på hvem som gjør hva hele tiden, og dette er en fantastisk løsning.
